# Frontend сервиса планирования производства

React + TypeScript + Vite клиент к backend-API планировщика. Главный экран — не абстрактный график
по участкам, а прямой ответ на вопрос «кто чем загружен»: сетка сотрудник × день недели.

Собран и проверен: `tsc -b` компилируется без ошибок, `npm run build` собирает прод-бандл,
JSON-ответы backend вживую сверены с TypeScript-типами (`src/api/types.ts`) — поля совпадают 1:1.

## Стек

React 19, TypeScript, Vite. Без сторонних UI-китов и стейт-менеджеров — на этом масштабе
достаточно `fetch` + `useState`/`useEffect` и одного React Context для авторизации.

## Структура

```
src/
  api/
    client.ts        — fetch-обёртка с JWT и обработкой ошибок
    endpoints.ts      — типизированные функции под каждый маршрут backend
    types.ts          — типы, сверенные с реальными ответами API
  context/
    AuthContext.tsx   — логин, текущий пользователь, роль
  components/
    Shell.tsx             — шапка + таббар
    ShopsPanel.tsx         — цеха и календарь смен
    OrderBuilderPanel.tsx  — конструктор заказа: изделия+количество, плюс отдельные операции вручную
    CellDetailPanel.tsx    — детали ячейки доски: отчёт по факту, ручное переназначение работнику
    LiveTimer.tsx          — тикающий секундомер для замера времени
  views/
    LoginView.tsx
    WorkersBoardView.tsx  — главный экран: сетка «сотрудник × день», формирование нарядов на неделю
    OrdersView.tsx        — список заказов, конструктор нового заказа
    CatalogView.tsx       — справочник норм + замеры времени + изделия
    AnalyticsView.tsx     — срывы сроков, загрузка цехов, эффективность работников
```

## Важное упрощение (сознательное)

Вместо drag-and-drop по абстрактным часам — клик по ячейке (работник × день) открывает список его
операций на этот день, где можно отчитаться по факту или явным выбором из списка перекинуть операцию
на другого работника того же участка. Функционально то же самое действие управления распределением,
но через явную форму, а не через отслеживание мыши по временной шкале — это и была основная жалоба
на предыдущую версию доски («непонятно, кто чем загружен»).


## Локальный запуск

```bash
cp .env.example .env
# укажите VITE_API_URL, если backend не на localhost:3000

npm install
npm run dev        # http://localhost:5173
```

Backend должен быть запущен и засеян демо-данными (см. README backend, `npm run db:seed`).
Войдите как dispatcher@example.com с паролем из SEED_ADMIN_PASSWORD backend `.env`.

## Роли в интерфейсе

- ADMIN / DISPATCHER — конструктор заказа, срочный заказ, закрепление любых операций, проекты, цеха
- NORMIROVSHIK — справочник операций, модификации, замеры времени; график только на просмотр
- SHOP_MASTER — видит график и справочник; закрепляет операции только на ресурсах своего цеха
  (интерфейс не блокирует клик заранее — покажет причину отказа, если сервер вернёт 403, так виден
  реальный, а не мнимый, источник ограничения)

## Продакшен-сборка и деплой на Timeweb Cloud

```bash
npm run build     # dist/ — статические файлы
```

Результат — чистый статический сайт, поэтому проще всего разместить его как статическое приложение
Timeweb Cloud Apps (или контейнерным способом, обернув dist/ в лёгкий nginx-образ, как ниже).
Перед сборкой обязательно задайте VITE_API_URL на реальный адрес задеплоенного backend — Vite
встраивает переменную в бандл на этапе сборки, поэтому это нужно сделать до npm run build.

Пример Dockerfile для контейнерного варианта:

```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
ARG VITE_API_URL
ENV VITE_API_URL=$VITE_API_URL
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80
```

## Известные ограничения

- Клик-редактор закрепления вместо drag-and-drop (см. выше)
- Нет real-time обновления графика между пользователями — после действия другого диспетчера график
  обновится только при вашем следующем действии или обновлении страницы
- Нет создания пользователей из интерфейса — только через POST /api/auth/users (curl/Postman),
  оставлено вне UI как редкая административная операция
